# Лабораторная №2

## Пункт 1. Конвертация между цветовыми пространствами

### Зачем нужны HSV и HSL

В `RGB` цвет описывается смесью трёх каналов, но человеку часто удобнее думать так:

- какой это цвет;
- насколько он насыщенный;
- насколько он яркий или светлый.

Поэтому используют другие пространства.

#### HSV

`HSV` расшифровывается как:

- `H` (`Hue`) — цветовой тон;
- `S` (`Saturation`) — насыщенность;
- `V` (`Value`) — яркость.

#### HSL

`HSL` расшифровывается как:

- `H` (`Hue`) — цветовой тон;
- `S` (`Saturation`) — насыщенность;
- `L` (`Lightness`) — светлота.

Важно понимать:

- `Hue` в обоих пространствах считается одинаково;
- различие между `HSV` и `HSL` в том, как именно определяется второй и третий смысловой параметр.

---

### Интуиция по компонентам

#### Hue

`Hue` отвечает на вопрос: какой это цвет на цветовом круге.

Примеры:

- `0` градусов — красный;
- `60` градусов — жёлтый;
- `120` градусов — зелёный;
- `180` градусов — голубой;
- `240` градусов — синий;
- `300` градусов — пурпурный.

#### Saturation

`Saturation` показывает, насколько цвет насыщенный.

- `S = 0` означает, что цвет серый;
- чем больше `S`, тем «чище» цвет.

#### Value

В `HSV` компонент `Value` отвечает за яркость.

- `0` означает чёрный;
- `1` означает максимально яркий цвет.

#### Lightness

В `HSL` компонент `Lightness` показывает светлоту.

- `0` — чёрный;
- `0.5` — условно «середина»;
- `1` — белый.

Из-за этого чистые яркие цвета вроде красного, зелёного и синего в `HSL` часто имеют `L = 0.5`.

---

### Базовые величины: cmax, cmin и delta

Для каждого пикселя после нормализации берём:

- `r`, `g`, `b`
- `cmax = max(r, g, b)`
- `cmin = min(r, g, b)`
- `delta = cmax - cmin`

Эти три величины используются почти во всех формулах.

#### Что означает cmax

Это самый сильный из трёх каналов. Он помогает понять:

- насколько пиксель яркий;
- в какой сектор цветового круга он попадает.

#### Что означает cmin

Это самый слабый канал.

#### Что означает delta

`delta` показывает, есть ли у пикселя вообще выраженный цвет.

Если:

- `delta = 0`

то это означает:

- `r = g = b`

Значит, цвет серый. У такого пикселя:

- нет определённого тона;
- насыщенность должна быть нулевой.

---

### Формула для Hue

`Hue` зависит от того, какой канал максимальный.

Если максимумом является `r`:

`H = 60 * (((g - b) / delta) mod 6)`

Если максимумом является `g`:

`H = 60 * (((b - r) / delta) + 2)`

Если максимумом является `b`:

`H = 60 * (((r - g) / delta) + 4)`

Почему формула разная:

- цветовой круг разбит на сектора;
- разные каналы соответствуют разным участкам круга;
- сначала определяется сектор, а потом точное положение внутри него.

Если `delta = 0`, то `Hue` формально не определён, но на практике обычно ставят `0`.

---

### Формулы для HSV

После вычисления `cmax`, `cmin` и `delta`:

- `V = cmax`
- `S = 0`, если `cmax = 0`
- иначе `S = delta / cmax`

Почему это работает:

- если пиксель чёрный, то `cmax = 0`, и насыщенность не имеет смысла, её задают как `0`;
- чем сильнее разница между максимальным и минимальным каналом, тем выше насыщенность.

---

### Формулы для HSL

После вычисления `cmax`, `cmin` и `delta`:

- `L = (cmax + cmin) / 2`
- `S = 0`, если `delta = 0`
- иначе `S = delta / (1 - |2L - 1|)`

Почему здесь другая формула:

- `HSL` старается выразить цвет через положение между чёрным и белым;
- поэтому светлота зависит сразу от максимального и минимального канала;
- насыщенность определяется иначе, чем в `HSV`.

---

### Почему в визуализации канал H выглядит странно, но это нормально

При показе только канала `H` через `Matplotlib` важно помнить:

- сам канал `H` хранится в градусах `0..360`;
- для отображения с `cmap='hsv'` его лучше делить на `360`, чтобы получить диапазон `0..1`.

Иначе `Matplotlib` будет автоматически нормализовать значения по текущей картинке, и изображение может выглядеть непривычно.

---

### Почему канал L в HSL выглядит серым

Это корректно.

Мы показываем не весь цвет, а только один канал `L`, то есть светлоту.

Если у нескольких разных цветов одинаковая светлота, то на картинке они будут иметь одинаковый оттенок серого.

Например, чистые:

- красный;
- зелёный;
- синий;
- жёлтый;
- голубой;
- пурпурный

часто имеют `L = 0.5`, поэтому на изображении канала `L` они все выглядят одинаково средне-серыми.

## Пункт 2. Blending изображений

Cмешивание двух изображений.

Обычно используется формула:

`result = alpha * image1 + beta * image2 + gamma`

где:

- `alpha` — вес первого изображения;
- `beta` — вес второго изображения;
- `gamma` — дополнительный сдвиг яркости.

Именно такую формулу использует и `OpenCV` в функции `cv2.addWeighted`.

---

### Что означает blending

Blending — это линейная комбинация двух изображений.

Если говорить просто:

- берём пиксель из первого изображения;
- берём пиксель из второго изображения;
- умножаем каждый на свой коэффициент;
- складываем;
- при необходимости добавляем константу `gamma`.

Это делается для каждого пикселя и для каждого канала.

Если изображения цветные, то формула применяется отдельно к:

- красному каналу;
- зелёному каналу;
- синему каналу.

---

### Интуиция по параметрам alpha, beta и gamma

#### alpha

Параметр `alpha` отвечает за вклад первого изображения.

Например:

- `alpha = 1.0` — первое изображение берётся полностью;
- `alpha = 0.5` — вклад первого изображения уменьшается вдвое;
- `alpha = 0.0` — первое изображение вообще не влияет.

#### beta

Параметр `beta` отвечает за вклад второго изображения.

Примеры аналогичны:

- `beta = 1.0` — второе изображение берётся полностью;
- `beta = 0.3` — второе изображение влияет слабее;
- `beta = 0.0` — второе изображение игнорируется.

#### gamma

`gamma` — это просто прибавка ко всем пикселям после смешивания.

То есть:

- если `gamma > 0`, изображение становится светлее;
- если `gamma < 0`, изображение становится темнее;
- если `gamma = 0`, дополнительного сдвига нет.

---

### Что происходит на уровне одного пикселя

Пусть есть два пикселя:

- из первого изображения: `[200, 100, 50]`
- из второго изображения: `[20, 180, 220]`

И параметры:

- `alpha = 0.7`
- `beta = 0.3`
- `gamma = 10`

Тогда новый пиксель считается так:

- `R = 0.7 * 200 + 0.3 * 20 + 10`
- `G = 0.7 * 100 + 0.3 * 180 + 10`
- `B = 0.7 * 50 + 0.3 * 220 + 10`

То есть каждый канал смешивается независимо.

## Пункт 3. Пороговая обработка

Для этого пункта нужно реализовать:

- бинарную;
- обратную бинарную;
- зануляющую (`to zero`);
- обратную зануляющую;
- truncated;
- гауссовскую адаптивную пороговую обработку.

Пороговая обработка обычно применяется к изображению в оттенках серого.

---

### Что такое пороговая обработка

Идея очень простая:

- у каждого пикселя есть численное значение яркости;
- мы сравниваем его с порогом;
- в зависимости от результата выдаём новое значение.

Если изображение `uint8`, яркость пикселя лежит в диапазоне:

- от `0` до `255`.

Например, при пороге `127`:

- всё, что больше `127`, считается светлым;
- всё, что меньше или равно `127`, считается тёмным.

---

## Почему сначала переводят изображение в grayscale

Пороговая обработка обычно работает не с тремя каналами `RGB`, а с одним каналом яркости.

То есть сначала цветное изображение переводится в grayscale.

После этого каждый пиксель описывается одним числом, и сравнение с порогом становится однозначным.

---

## Бинарная пороговая обработка

Для `binary threshold` формула такая:

- если `src(x, y) > T`, то результат равен `max_value`;
- иначе результат равен `0`.

Обычно:

- `T = 127`
- `max_value = 255`

То есть изображение превращается в чёрно-белое.

---

## Обратная бинарная

Для `inverse binary threshold` всё наоборот:

- если `src(x, y) > T`, то результат равен `0`;
- иначе результат равен `max_value`.

Это просто инверсия обычного бинарного порога.

---

## Зануляющая (`to zero`)

Здесь пиксели, не прошедшие порог, зануляются:

- если `src(x, y) > T`, то пиксель сохраняется как есть;
- иначе становится `0`.

То есть яркие области сохраняются, а тёмные убираются.

---

## Обратная зануляющая

В `to zero inverse` наоборот:

- если `src(x, y) > T`, то становится `0`;
- иначе сохраняется исходное значение.

Здесь сохраняются тёмные области, а светлые убираются.

---

## Truncated threshold

В этом режиме:

- если `src(x, y) > T`, то результат становится равным `T`;
- иначе пиксель сохраняется как есть.

То есть слишком яркие пиксели «обрезаются» сверху.

---

## Что такое адаптивная пороговая обработка

Обычный порог использует одно число `T` для всего изображения.

Это плохо работает, если:

- часть изображения слишком тёмная;
- часть слишком светлая;
- освещение неравномерное.

Тогда используют адаптивный порог:

- для каждого пикселя порог считается отдельно;
- порог зависит от соседних пикселей.

---

## Гауссовская адаптивная пороговая обработка

В варианте `adaptive gaussian threshold` локальный порог считается как:

- взвешенная сумма соседних пикселей по гауссовскому окну;
- затем из неё вычитается константа `C`.

То есть:

- ближайшие соседи влияют сильнее;
- дальние соседи влияют слабее.

Это лучше, чем просто среднее, потому что центр окна получает больший вес.

---

## Что такое гауссовское окно

Это матрица весов, где:

- максимальный вес в центре;
- по мере удаления от центра веса уменьшаются.

Примерно так:

```text
1 2 1
2 4 2
1 2 1
```

Но в реальности значения рассчитываются по формуле Гаусса и потом нормализуются так, чтобы сумма всех весов была равна `1`.

---

## Как работает adaptive gaussian threshold

Для каждого пикселя:

1. берётся окрестность размера `block_size x block_size`;
2. к ней применяется гауссовское окно;
3. считается локальное взвешенное среднее;
4. из него вычитается `C`;
5. исходный пиксель сравнивается с этим локальным порогом.

Если пиксель больше локального порога:

- ставим `max_value`

иначе:

- ставим `0`.

---

## Зачем нужен block_size

`block_size` задаёт размер локальной области, по которой считается порог.

Важно:

- он должен быть нечётным;
- иначе у окна не будет центрального пикселя.

Маленькое окно:

- сильнее реагирует на локальные детали;
- может быть более шумным.

Большое окно:

- делает порог более сглаженным;
- лучше работает при плавном изменении освещения.

---

## Что такое фильтрация изображения

Фильтрация означает, что новое значение пикселя считается по соседним пикселям.

Обычно:

1. берётся небольшое окно вокруг текущего пикселя;
2. это окно комбинируется по некоторому правилу;
3. результат записывается в новый пиксель.

Если правило основано на матрице коэффициентов, такую матрицу называют:

- ядром;
- маской;
- kernel.

---

## Применение собственного фильтра

Собственный фильтр — это просто ядро, которое мы выбираем сами.

Например, можно взять ядро повышения резкости:

```text
 0 -1  0
-1  5 -1
 0 -1  0
```

Что оно делает:

- центральный пиксель усиливается;
- соседние немного вычитаются;
- изображение становится более резким.

Реализация сводится к свёртке изображения с ядром.

---

## Что такое свёртка

Свёртка — это операция, при которой:

- окно того же размера, что и ядро, накладывается на изображение;
- значения пикселей умножаются на коэффициенты ядра;
- все произведения суммируются.

Для каждого положения окна получается одно новое значение пикселя.

В `NumPy` это можно реализовать вручную через:

- padding изображения;
- проход по координатам;
- вычисление суммы `window * kernel`.

---

## Зачем нужен padding

Когда окно находится около границы изображения, часть соседей выходит за пределы картинки.

Поэтому перед свёрткой изображение обычно расширяют.

Популярные варианты:

- повторение крайних значений;
- отражение;
- заполнение нулями.

В ноутбуке используется повторение ближайшего края, чтобы поведение было ближе к `OpenCV`.

---

## Медианное размытие

Медианный фильтр работает не как обычная свёртка.

Для каждого окна:

- все значения внутри окна собираются в набор;
- они сортируются;
- центральному пикселю присваивается медиана.

Например, для значений:

`[10, 10, 11, 12, 200]`

медиана будет:

`11`

Главное свойство медианного фильтра:

- он хорошо убирает импульсный шум;
- при этом часто сохраняет границы лучше, чем обычное усреднение.

---

## Гауссовское размытие

Гауссовское размытие — это свёртка с гауссовским ядром.

Идея такая:

- максимальный вес находится в центре окна;
- ближайшие пиксели влияют сильнее;
- дальние пиксели влияют слабее.

Это даёт более естественное размытие, чем простое среднее.

---

## Как генерируется гауссовское ядро

Ядро строится по формуле Гаусса:

`G(x, y) = exp(-(x^2 + y^2) / (2 * sigma^2))`

Дальше все элементы ядра нормализуются так, чтобы их сумма была равна `1`.

Это важно, иначе изображение после фильтра может стать:

- слишком тёмным;
- слишком ярким.

---

## Что такое sigma

`sigma` управляет степенью размытия.

- маленькое `sigma` — слабое размытие;
- большое `sigma` — сильное размытие.

Если `kernel_size` увеличивается, обычно имеет смысл увеличить и `sigma`.

---

## Чем отличаются median и gaussian blur

`Median blur`:

- выбирает медиану;
- особенно полезен против шума типа «соль и перец».

`Gaussian blur`:

- делает взвешенное усреднение;
- хорошо подходит для общего сглаживания перед дальнейшей обработкой.

---

## Пункт 5. Фильтр Собеля

Фильтр Собеля используется для выделения границ.

Он оценивает, насколько быстро меняется яркость:

- по горизонтали;
- по вертикали.

Для этого обычно берут два ядра:

### Sobel X

```text
-1  0  1
-2  0  2
-1  0  1
```

### Sobel Y

```text
-1 -2 -1
 0  0  0
 1  2  1
```

Первое ядро чувствительно к вертикальным границам, второе — к горизонтальным.

---

## Что получается после Sobel

После применения фильтра получаются две карты:

- `Gx` — градиент по `x`;
- `Gy` — градиент по `y`.

Чтобы получить общую силу границы, обычно считают:

`G = sqrt(Gx^2 + Gy^2)`

После этого результат ограничивают диапазоном `0..255`.

## Что такое кумулятивная функция распределения

Если гистограмма показывает количество пикселей на каждом уровне, то кумулятивная функция распределения, или `CDF`, показывает накопленную сумму.

- `CDF[0]` — сколько пикселей имеют яркость `0`;

Формально:

`CDF[i] = hist[0] + hist[1] + ... + hist[i]`

1. считаем гистограмму;
2. считаем `CDF`;
3. нормализуем `CDF` в диапазон `0..255`;
4. используем получившуюся таблицу как отображение старых значений в новые.

То есть каждый пиксель заменяется новым значением по lookup-table.

---

## Почему нужно учитывать минимальное ненулевое значение CDF

Если просто нормализовать всю `CDF`, результат может быть некорректным, если часть низких яркостей вообще не встречается.

Поэтому обычно берут:

- минимальное ненулевое значение `CDF`;
- и нормализуют уже относительно него.

Это соответствует стандартной формуле эквализации.
